# CLAUDE.md — Art in LA

Project context for Claude Code. Read this first. The narrative changelog of how
the project got to its current state lives in `PROJECT_STATUS.md`.

## What this is

A community-maintained map + calendar of **museum and non-commercial art
institution** exhibitions and events across Los Angeles County. Static React +
Vite site on GitHub Pages, fed by Python scrapers that run on a daily GitHub
Actions cron and commit fresh JSON data.

- Repo: `github.com/birdofnofeather/art-in-la`, default branch `main`.
- Live site: `https://birdofnofeather.github.io/art-in-la/`
- Scope: **museums + institutions only.** Commercial galleries were intentionally
  removed (see below). Currently **48 scrapers**, **79 venues**
  (31 museum / 22 community / 17 alternative / 9 academic), ~45 venues produce events.

## Layout

```
public/data/      venues.json (hand-curated) + events.json, archive.json,
                  scraped_venues.json, warnings.json, health.json (ALL generated by CI)
scrapers/         base.py (BaseScraper + 6 strategies), registry.py (active list),
                  run_all.py (orchestrator), utils/, venues/<id>.py (one per venue)
scrapers/venues/_archived_galleries/   retired gallery scrapers (NOT imported)
src/              React app: App.jsx, components/, lib/ (data, filters, constants)
.github/workflows/ daily-scrape.yml (cron) + deploy.yml (Pages)
commit.ps1        one-command commit+push helper (Windows)
```

## Conventions and decisions (do not regress these)

### Scope: no galleries
Commercial galleries were removed. Their scrapers live in
`scrapers/venues/_archived_galleries/` (kept for reference, not imported by
`registry.py`), and gallery venues/events were purged from the data. Do not
re-add galleries unless the owner asks. The frontend taxonomy is exactly:
`museum | community | alternative | academic`.

### Dates & times: LA-local, never shifted
Everyone (venues + audience) is in Los Angeles. `scrapers/utils/dateparse.py:to_la_iso`
is the single source of truth, and `Event.to_dict()` runs every event through it.
On the frontend, `src/lib/filters.js:parseDate` parses the stored LA wall-clock
fields verbatim (no viewer-timezone conversion) so every visitor sees the same
times the venue's own site shows — do not replace it with bare `new Date(s)`:
- A real time keeps the source wall-clock; if the source tags it UTC, it is
  converted to the equivalent LA time (so it matches what the venue's site shows).
- A value at midnight in its own timezone is a **date-only** listing → returned as
  a bare `YYYY-MM-DD` (all-day). **Never fabricate a `00:00` / "12:00 AM" time.**
- There is no other timezone conversion anywhere.

### Recurring tours filtered
`scrapers/utils/recurring.py` drops standing recurring programmes, including
architecture / garden / gallery tours (e.g. Getty "Art, Architecture, and Garden
Tour"). Real one-off curator/exhibition tours are kept.

### HTTP
`scrapers/utils/http.py` sends a desktop-Chrome User-Agent because several venue
sites (Norton Simon, Armory) 403/429 a bot UA. Keep it browser-like.

### Parallel scraping
`run_all.py` scrapes venues concurrently via a thread pool (`SCRAPE_WORKERS` env,
default 8); a full run is ~27s. Output is deduped + sorted afterward, so order is
stable. Set `SCRAPE_WORKERS=1` to debug sequentially.

### Frontend tabs
Top nav: **Map / Events / Exhibitions / Venues / Archive**. Events and Exhibitions
are SEPARATE tabs (exhibitions are not nested under Events). Events have a "Today"
date preset; Exhibitions have On view now / Upcoming / All. `lib/data.js` de-dupes
venues by id on load (a duplicate id otherwise renders two map markers and breaks
the id-keyed detail lookup — this actually happened with SPARC).

### Generated data + git
`events/archive/scraped_venues/warnings.json` are rewritten daily by CI. `.gitattributes`
marks them `merge=ours` so pull/rebase auto-resolves them; line endings are LF.
`venues.json` is the only hand-curated data file.

## Bot-gated venues: headless-browser render path

Two venues serve an event-less shell (or a hard 429) to plain HTTP clients but
hand a real browser the full calendar once their anti-bot JS runs:
- `norton_simon` — Cloudflare; the SPA renders events client-side.
- `huntington` — Vercel "Security Checkpoint" (429) that **self-clears after a
  few seconds if you wait in-page** instead of re-navigating.

Both are now scraped via `scrapers/utils/render.py`, which runs
`render_cli.py` (headless Chromium / Playwright) in a **subprocess** — keeping
the thread-unfriendly browser work out of the scrape thread pool, the same way
the Getty scraper shells out to Node. The renderer is deliberately minimal:
spoof only `navigator.webdriver` + `window.chrome` (piling on more navigator
overrides made Vercel's checkpoint stick), wait the interstitial out in-page,
then return the populated HTML. CI installs it via `requirements-render.txt`
+ `playwright install --with-deps chromium`. If Playwright is absent (a plain
local run), these two scrapers no-op and the rest of the pipeline is unaffected.

## Running it

```bash
# Frontend
npm install && npm run dev            # http://localhost:5173
npm run build

# Scrapers
pip install -r scrapers/requirements.txt
python -m scrapers.run_all            # all venues, writes public/data/*.json
python -m scrapers.run_all --only ica_la --dry-run   # one venue, no write

# Bot-gated venues (Norton Simon, Huntington) — needs the headless browser:
pip install -r scrapers/requirements-render.txt && python -m playwright install chromium
python -m scrapers.run_all --only norton_simon,huntington --dry-run
```

Commit + push (Windows): `.\commit.ps1 "message"` (validates JSON, stages,
commits, rebases, pushes).
